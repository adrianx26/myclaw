## 2024-05-14 - Optimize config loading via mtime check and caching
**Learning:** Frequent file reads (`read_text()`) and existence checks (`exists()`) on configuration files within often-called functions significantly impact performance, causing unnecessary filesystem overhead. Avoiding redundant directory creation steps during steady-state execution further reduces syscalls.
**Action:** Implement module-level caching and use `st_mtime` via a `try/except FileNotFoundError` block to intelligently update cache state only when the underlying configuration file changes. Conditionally manage directory generation based on the initialized cache state to bypass redundant `mkdir` executions. Utilize `copy.deepcopy()` to protect the cache from caller mutation and ensure modifications are saved to disk *before* updating the cache to prevent unsynchronized state on write failures.
