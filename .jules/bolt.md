## 2024-05-24 - File IO Performance Bottleneck in Config Loading
**Learning:** Frequent application of `pathlib.Path.exists()` and `.mkdir()` in heavily used configuration loading functions (`myclaw/config.py`) creates unnecessary filesystem syscall overhead. Checking `exists()` is essentially an extra `stat` call that can be avoided by just trying the operation and handling exceptions.
**Action:** Implement module-level memory caching for configurations mapped to `stat().st_mtime`. Use a `try/except FileNotFoundError` block to handle file reads instead of using `exists()`, allowing skipping overhead entirely if the cache is valid, and conditionally bypass directory initializations (`mkdir`) only when the cache is uninitialized (`None`). Also `copy.deepcopy()` is needed to ensure we don't return mutable references to the internal cache which could cause unintended state mutations.
