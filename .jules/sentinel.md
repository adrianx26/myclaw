## 2025-03-08 - Prevent Path Traversal in File Operations
**Vulnerability:** File operations (`read_file`, `write_file`) lacked proper validation of user-provided paths, allowing traversal outside the designated `WORKSPACE` directory using absolute paths or `../`. In addition, catching arbitrary exceptions and casting them to strings leaked sensitive directory structures.
**Learning:** `pathlib.Path` is susceptible to path traversal if `.resolve()` is not used and combined with an `.is_relative_to()` check. Simply using the `/` operator with an absolute right-hand operand insecurely defaults to the absolute path. Error messages must fail securely with generic text to prevent information disclosure.
**Prevention:** Always use `target_path = (BASE_DIR / input_path).resolve()` followed by `if not target_path.is_relative_to(BASE_DIR.resolve()):` before performing file I/O operations. Ensure exception blocks return safe, generic error messages like `"Error: Access denied"`.
