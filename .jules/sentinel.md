## 2024-05-24 - [Path Traversal in myclaw/tools.py Workspace Tooling]
**Vulnerability:** The `read_file` and `write_file` functions in `myclaw/tools.py` constructed file paths simply by using `WORKSPACE / path` without verifying the resulting path. If a user provided a path such as `../` or an absolute path (e.g. `/etc/passwd`), the `/` operator in `pathlib` combined with `.read_text()` / `.write_text()` allowed arbitrary file access outside the intended `WORKSPACE` directory. Also, catching `Exception as e` and returning `f"Error: {e}"` leaked internal file structures.
**Learning:** Using `pathlib` operator `/` does not automatically normalize paths or restrict access. If the right operand is an absolute path or contains `..`, it can escape the left operand's directory. Security functions intended to sandbox file operations must actively check boundaries. Additionally, stringifying arbitrary file system exceptions risks leaking internal directory structures or existence.
**Prevention:** Always normalize and resolve paths when accepting user input for file operations using `.resolve()`. Validate that the resulting target path is constrained within the allowed base directory using `target_path.is_relative_to(WORKSPACE.resolve())`. Fail securely by using generic error messages instead of leaking the stringified exception.
