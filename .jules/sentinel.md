## 2024-05-24 - [Path Traversal & Info Leak in Workspace Tools]
**Vulnerability:** The tools `read_file` and `write_file` did not validate paths properly, allowing directory traversal and arbitrary file access via `../` sequences or absolute paths outside the workspace directory. Furthermore, exception blocks leaked stringified errors `f"Error: {e}"` revealing file system details.
**Learning:** Using Python's `pathlib` for file operations is secure *only if* paths are normalized using `.resolve()` and bound within the expected boundary using `.is_relative_to()`. The `/` operator does not automatically normalize paths and behaves insecurely when absolute paths are passed as the right operand. Additionally, raw exception leakage provides system reconnaissance capabilities to attackers.
**Prevention:** Always use `path = (base_dir / input_path).resolve()` followed by `if not path.is_relative_to(base_dir.resolve()): return "Error: Access denied"`. Consistently return generic error messages (e.g. "Error: Access denied" or "Error: Command execution failed") in exception blocks rather than exposing stringified errors or stack traces.
