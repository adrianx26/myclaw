## 2024-05-18 - Prevent Path Traversal in File Operations
**Vulnerability:** The application used `(WORKSPACE / path).read_text()` and `(WORKSPACE / path).write_text(content)` in `myclaw/tools.py` without resolving or restricting paths. Because the `path` argument was completely unsanitized, an absolute path starting with `/` or traversal components (`../`) would allow agents to escape the restricted WORKSPACE directory. Furthermore, the `except Exception as e: return f"Error: {e}"` block directly leaked file structure paths on failures (e.g., FileNotFoundError).
**Learning:** Python's `pathlib.Path` `/` operator automatically discards the left side of the operator if the right side string is an absolute path. Thus, simple concatenation is insecure against malicious absolute paths. Additionally, stringifying the exception directly leaks potentially sensitive context on why an action failed.
**Prevention:** Always restrict dynamically joined path strings by first obtaining the fully normalized absolute path using `.resolve()` and then verifying the restriction boundary using `.is_relative_to()`. Finally, always catch exceptions and return generic, sanitized errors (e.g., "Error: Access denied") to avoid stack trace/existence disclosure.
