## 2024-05-24 - Path Traversal & Information Disclosure in Workspace Tools
**Vulnerability:** The `read_file` and `write_file` functions in `myclaw/tools.py` concatenated user input paths with the `WORKSPACE` directory without normalizing and validating the final path, allowing arbitrary file reading/writing outside the intended workspace via `../../` patterns or absolute path prefixes. Furthermore, exceptions were stringified and returned to the user, potentially leaking system internals (e.g. stack traces, directory structures).
**Learning:** Python's `pathlib.Path` `/` operator acts insecurely when the right-hand operand is an absolute path (it discards the left operand). The paths must be resolved and checked against the resolved `WORKSPACE` directory using `is_relative_to()`. Catching and stringifying all exceptions in file operations leaks sensitive filesystem details.
**Prevention:** Always normalize paths using `.resolve()` and verify the target path stays within boundaries via `.is_relative_to(ROOT.resolve())`. Use explicit error handling (e.g., catching `FileNotFoundError`) and return generic error messages (e.g., "Error: Access denied") for general exceptions to prevent information disclosure.
