## 2024-05-24 - Python Pathlib Path Traversal Vulnerability
**Vulnerability:** Path traversal (arbitrary file read/write) in `myclaw/tools.py`. The `/` operator in `pathlib` with an absolute path on the right-hand side (`WORKSPACE / "/etc/passwd"`) evaluates to the absolute path (`/etc/passwd`), entirely bypassing the `WORKSPACE` prefix. Additionally, paths like `WORKSPACE / "../../../etc/passwd"` were evaluating but without boundary checking. Error messages returned stringified exceptions which leaked path or file existence information.
**Learning:** Using `pathlib`'s `/` operator is not inherently secure against absolute path overrides or parent directory traversal `../`. String `startswith` validation on `pathlib.Path` objects is insecure. Exception blocks on file I/O operations can inadvertently leak sensitive system paths when casting `Exception as e` to a string.
**Prevention:** Always use `path.resolve()` on constructed paths and immediately verify boundaries using `.is_relative_to(BASE_DIR.resolve())` before executing any file operations. In exceptions related to files, return generic, fixed error messages like `"Error: Access denied"` rather than returning specific failure details.
