## 2024-03-16 - Path Traversal in File I/O Tools
**Vulnerability:** The `read_file` and `write_file` functions in `myclaw/tools.py` constructed file paths using string concatenation without validation or resolving, allowing arbitrary file read/write across the entire filesystem using path traversal (e.g., `../../../../etc/shadow`). Furthermore, exception handlers exposed the full paths and system error strings.
**Learning:** Naive path concatenation using `/` with `Path` objects does not normalize absolute paths or path traversal constructs (`..`). The right-side argument to `/` can even override the left if it's an absolute path. Security relies entirely on proper path resolution and isolation validation.
**Prevention:** Always use `.resolve()` to normalize paths and eliminate traversal segments *before* checking bounds. Use `.is_relative_to(BASE_DIR.resolve())` to ensure the target remains within the allowed directory tree. Never return raw exception strings in failure handlers for I/O operations, as they leak internal structure and presence of files.
