## 2024-03-11 - Prevent Path Traversal in File Operations
**Vulnerability:** The application was vulnerable to path traversal due to appending user input directly to a base directory (`WORKSPACE / path`), which, when containing `..` or absolute paths, allowed attackers to read/write arbitrary files outside the workspace directory. Furthermore, the generic catch block `except Exception as e:` leaked internal directory structure and existence of files in its stringified format.
**Learning:** `Pathlib`'s `/` operator does not automatically normalize paths and behaves insecurely when the right-hand side is an absolute path. Any user input combined with a base path must be sanitized.
**Prevention:** Use `.resolve()` to normalize the complete path, followed by `.is_relative_to(BASE_DIR.resolve())` to ensure the resolved path truly resides within the target directory. Additionally, do not stringify exceptions when handling file operations, but rather return a generic "Access denied" error.
