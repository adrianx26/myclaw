## 2024-05-14 - Path Traversal Vulnerability with pathlib `/` Operator
**Vulnerability:** Arbitrary file read/write due to path traversal and absolute path handling. Unsafe use of `pathlib` operator `WORKSPACE / path` allowed paths like `/etc/passwd` or `../../secret.txt` to bypass intended directory restrictions. Exception handling leaked system errors.
**Learning:** In Python's `pathlib`, using the `/` operator with an absolute path on the right side replaces the left side entirely. Also, `pathlib` doesn't automatically normalize `..` segments unless `.resolve()` is used. String prefix matching (e.g., `startswith`) is insufficient for secure path confinement. Generic error messages must be used to prevent leaking directory structure details or file existence when exceptions are raised.
**Prevention:** Always use `.resolve()` on the combined path to normalize it (evaluating symlinks and `..`), and then verify it falls within the intended root directory using `.is_relative_to(root_path.resolve())` before performing any operations. Use generic error messages (e.g., "Error: Access denied") in exception handlers.
