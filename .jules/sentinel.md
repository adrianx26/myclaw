## 2024-03-10 - Path Traversal in pathlib Division
**Vulnerability:** The application was vulnerable to path traversal because the division operator (`/`) in `pathlib` interprets right-hand string operands starting with `/` as absolute paths, and standard string concatenation allows traversal via `..`. This allows accessing files outside the designated `WORKSPACE`.
**Learning:** `pathlib.Path(dir) / user_input` does not prevent path traversal if `user_input` contains absolute paths or traversal sequences (`../`). Furthermore, simply using `.startswith()` on strings is inadequate for path isolation checks as it is prone to bypassing and fails to canonicalize paths.
**Prevention:** Always use `.resolve()` to normalize and canonicalize user-provided paths before security checks. Then, use `.is_relative_to(ROOT.resolve())` to securely enforce that the resulting path is constrained within the intended directory bounds. Additionally, catch and mask exceptions with generic errors (like "Access denied") instead of leaking raw exception strings (e.g. `str(e)`) to prevent disclosing file paths or internal server structure.
