## 2024-05-23 - Path Traversal Vulnerability in File Tools
**Vulnerability:** The `read_file` and `write_file` tools in `myclaw/tools.py` use `WORKSPACE / path` to construct file paths. In Python's `pathlib`, if `path` is an absolute path (starts with `/`), it completely overrides the left-hand side (`WORKSPACE`), allowing read/write access to any file on the system (e.g., `read_file("/etc/passwd")`). Furthermore, `../` traversal sequences were not blocked, allowing escaping the workspace.
**Learning:** Python `pathlib.Path` `/` operator behaves insecurely with absolute right-hand operands. The lack of `.resolve()` and boundary checking (`.is_relative_to()`) creates a critical path traversal and arbitrary file access vulnerability.
**Prevention:** Always sanitize paths before appending to a base directory (e.g., `str(path).lstrip('/')`), use `.resolve()` to eliminate `../` components, and explicitly verify the resolved path remains within the intended boundary using `.is_relative_to()`. Do not leak real paths or exceptions in error messages.
